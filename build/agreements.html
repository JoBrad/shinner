    POST /agreements

    Publish an agreement

    Publish an agreement with information about signatories

    Body parameter

    {
  "publisher": "publisher@example.com",
  "draft": true,
  "field_map": {},
  "signature_map": {
    "primary": "owner@example.com"
  },
  "email_options": {}
}

    Parameters

    Parameter In Type Required Description
    body body PublishRequest true The template request information.
    » publisher body string true The email address of the publisher of this agreement. If the "publisherId" is specified, it will be used instead.
    » draft body boolean true No description
    » field_map body FieldMap true No description
    » signature_map body object true The key:value map that specifies signatories (by email address) that will be associated with the signing document signature placeholders.
    » email_options body EmailOptions false No description

    Example responses

    {
  "agreement_status": {},
  "launch_urls": {},
  "published_documents": {},
  "field_map": {}
}

    Responses

    Status Meaning Description Schema
    200 OK Agreement Successfully published. PublishResponse

    Update agreement

    Update a single agreement.

    Body parameter

    {
  "agreement_status": {},
  "launch_urls": {},
  "published_documents": {},
  "field_map": {}
}

    Parameters

    Parameter In Type Required Description
    id path string true The unique ID of the agreement
    body body PublishResponse true Update action performed on a single agreement.
    » agreement_id body string false Agreement ID
    » agreement_status body AgreementStatus false No description
    » launch_urls body LaunchUrls false No description
    » published_documents body PublishedDocuments false No description
    » field_map body FieldMap false No description

    Responses

    Status Meaning Description Schema
    204 No Content The resource was updated successfully. None

    AuthResponse

    {}

    Properties

    Name Type Required Description
    access_token string false The token that should be used on subsequent calls
    application_name string false The GUID of the application
    api_product_list string false A list of API products that the user has access to
    api_product_list_json string false A valid JSON list of API products that the user has access to
    client_id string false The unique identifier used to request the token
    developer.email string false The email address on file for the user that requested the token
    expires_in number false How long the token will remain valid, in seconds after issued_at
    issued_at number false The timestamp when the token was issued, as a UNIX epoch
    organization_name string false The organization associated with the user who requested the token
    refresh_count number false How many times this token has been refreshed
    refresh_token_expires_in number false If this is a refresh response, how long the refreshed token will remain valid, in seconds
    scope string false The scope granted by this token
    status string false The status of the token
    token_type string false The type of token that was issued
