    Payments v1.0.0

    Scroll down for code samples, example requests and responses. Select a language for code samples from the tabs above or the mobile navigation menu.

    The Payments API provides methods for charging customers, refunding money to customers, and tokenizing payment methods to be used for charges or refunds.

    Base URLs:

    • https://test-api.i3verticals.com/v1

    • https://api.i3verticals.com/v1

    Terms of service Email: i3 Vertical's implementation team Web: i3 Vertical's implementation team License: Apache 2.0

    Authentication

    • API Key
      • Parameter Name: Authorization, in: header. The value Bearer access_token_value where access_token_value is the value of the access_token returned by the CreateBearerToken call

    CreateCardToken

    Create a secure token for a credit card

    Body parameter

    {
  "card_number": 4111111111111111,
  "cvv": 123,
  "expiration_date": 1222,
  "first_name": "John",
  "last_name": "Doe",
  "company_name": "Joe's Bait Shop",
  "address1": "500 Oak Street",
  "address2": "Unit 303",
  "city": "Saint Cloud",
  "state": "MN",
  "postal_code": 56304,
  "country": "United States",
  "reason": "Recurring",
  "token_expiration_date": "1/1/2019"
}

    Parameters

    Parameter In Type Required Description
    Authorization header string true The bearer token created when the user authenticated into the system.
    body body CreateCardToken false No description
    » card_number body string true The credit card number. Required for charges using a card that is not stored.
    » cvv body string false The 3 or 4 digit Card Verification Value. This parameter may be required depending on the merchant's setup.
    » expiration_date body string true The date that the card expires, in MM/YY format.
    » first_name body string true The first name of the customer as it appears on the payment type.
    » last_name body string true The family or surname of the customer as it appears on the payment type.
    » company_name body string false The name of the company, if this payment type is for a corporate entity.
    » address1 body string true The billing address associated with the payment type. Used for address verification (AVS) if it is enabled for the merchant.
    » address2 body string false The second address line associated with the payment type, if present. Used for address verification (AVS) if it is enabled for the merchant.
    » city body string true The city associated with the account.
    » state body string true The 2-digit abbreviation of the state associated with the account.
    » postal_code body string true The postal code associated with the account, without spaces or hyphens (if 5+4 format)
    » country body string true Customer's country of residence.
    » reason body string true The reason used when submitting the payment to the processor.
    » token_expiration_date body string false An optional date in the format MM/DD/YYYY indicating when the token should expire.

    Enumerated Values

    Parameter Value
    » reason OnFile
    » reason Recurring
    » reason Installment
